Overview

Dasana is a subscription-based mobile application primarily established in Mexico with ambitions to expand to Spain. It is oriented towards individuals looking to lead a healthy lifestyle, offering a wide range of content including recipes, exercise routines, meditations, and other valuable resources.

In the redesign process, the main objective was to revitalize the application as its previous design was perceived as outdated and not aligned with the company's current image. I aimed to modernize the interface by removing old features, incorporating new ones, and enhancing the overall user experience.

Research

To achieve an effective redesign, I conducted thorough research, analyzing popular platforms and applications among real users and through research. My goal was to avoid unnecessary changes and leverage existing strengths so that users would feel comfortable and not confused when using the application.

Findings:

  • Login functionality had significant shortcomings compared to popular applications.
  • Previous design was developed in an improvised manner, resulting in an outdated appearance. A more cohesive design approach was needed to create an attractive and consistent visual identity..
  • Existing flows for meditations and exercises were outdated and contained unnecessary steps. It was essential to rethink these flows to make them more efficient and appealing to users.
  • Navigation bar structure was poorly constructed.
  • All account options and settings were grouped on a single screen, creating an unfriendly and overwhelming experience for users.

Defining the Solution

Implemented the following improvements:
  • Proposed features such as password recovery and password visibility while typing. I also allowed login and registration through other accounts for a smoother experience.
  • Adopted a cleaner and more modern design, similar to current applications, highlighting key features with visually appealing design elements.
  • Established a new flow, presenting a daily meditation with details and a set of exercises for the week.
  • Restructured the content for simpler and organized navigation, making it easier to find relevant information.
  • Created additional sections on the profile screen for a more straightforward and detailed user experience.

Wireframing

Given the tight deadline of approximately 2 weeks for the redesign, instead of creating grayscale wireframes, I chose to analyze the existing flow and use it as a basis to create a completely new one.

Establishing the Visual Design

In the process of establishing the visual design for the new interface, I selected orange as the primary color in line with the brand's identity. I used various shades of orange, primarily against a white background. I also reorganized the buttons, making them larger for a more intuitive view and an improved visual presentation of content, thus simplifying the user experience.

High Fidelity Prototype

Maintained constant communication with the client to ensure that the creative directions aligned with their expectations. Despite having initial creative freedom, we achieved the following progress:
  • Introduced a splash screen to enhance the app's loading experience, along with registration and login options via Facebook, Google, and Apple.
  • Instead of using generic buttons for all functions, I optimized the organization of information on the home page by removing unnecessary elements and improving the overall aesthetics.
  • Redesigned the navigation bar, including shortcuts to meditations and exercises, and adjusted the business model to be more intuitive and familiar to the majority of users.
  • On the profile screen, I restructured and reorganized the content for a more cohesive and user-friendly experience.

Conclusions and Learnings

Redefine the flow of the application proved to be a challenge centered around logic and understanding how to create an effective flow to enhance the user experience.

Working on a redesign with time constraints and complete creative freedom presented a challenge. It's common for users to feel uncomfortable with changes initially, but positive feedback and an increase in users suggest that the project has been successful.

It's important to note that the design presented in this case study differs from the current version of the application due to various factors that have prevented its full implementation.
